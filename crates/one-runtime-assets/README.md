# Assets

Static assets, generated asset handles, and build output.

## Structure

- `images/` source images and icons
- `dist/` generated frontend assets such as `tailwind.css` and WASM bindings
- `input.css` Tailwind entrypoint
- `build.rs` typed asset generation with `cache-busters`
- `src/` crate entrypoints and re-exports

## Workflow

- Run `just tailwind` when CSS assets need rebuilding.
- Tailwind output is written to `dist/tailwind.css`.
- `build.rs` watches `images/` and `dist/`, then generates typed asset handles.
- Use asset references through `one_runtime_assets::files::*`.

## Rules

- Keep source images in `images/`.
- Treat `dist/` as generated output.
- Do not hand-write asset handle code in `src/`; it is generated by `build.rs`.
