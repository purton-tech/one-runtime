How connections work

One Runtime securely brokers OAuth for your product. You configure providers once, then use the One Runtime API to send your users through hosted connection links whenever access is needed.

Managed or customer-managed

Choose whether consent shows One Runtime or your own Google app.

Scoped and auditable

Every provider grant and API action is constrained by configured scopes and written to the audit log.

Revocable

Disable a provider, rotate credentials, or revoke access whenever your security posture changes.
