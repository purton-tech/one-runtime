Connect Gmail

Configure how One Runtime should broker Gmail authorization for your product before you generate connection links for end users.

Provider: Google

What One Runtime will do

Host the user authorization flow

Generate the Google redirect, exchange tokens, and store connection status for your project.

Enforce scoped access

Only the scopes you enable for Gmail will be requested and used for downstream API actions.

Record audit events

Every consent start, token exchange, refresh, and provider API call can be reviewed in the activity log.

What One Runtime will not do

  • Request broader Gmail access than your project configuration allows
  • Expose your Google client secret when you choose customer-managed OAuth
  • Prevent you from revoking tokens or disabling Gmail for your product
